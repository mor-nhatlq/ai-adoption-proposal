Quy Trình Testing Với AI — 5 Bước

AI là content generator (viết testcase). Tester là quality gate (review + execute). Không build agent tự động execute trong giai đoạn này — tập trung tăng tốc viết testcase & nâng chất lượng coverage.

OwnerQA Lead
Duration1 tuần build · 1 ngày workshop
Scope5 bước · 3 quality gates
PrincipleYAGNI · KISS · Human-in-the-loop
01 · Nguyên Tắc

AI viết — Tester quyết

Trong giai đoạn này, AI đóng vai trò force multiplier cho việc viết testcase. Không để AI tự chạy test & tự claim pass — mọi kết quả test phải có evidence người thật (screenshot, log, video).

Core Principle

AI: brainstorm strategy, break-down plan, viết testcase theo template. Tester: approve strategy, review testcase, execute & attach evidence. Mỗi bước AI làm đều có Tester đứng sau kiểm soát — không agent nào tự "done" mà không qua human gate.

02 · Workflow

Flow 5 bước — 3 gates

Từ requirements → test report. 3 bước đầu AI hỗ trợ (với Tester điều hướng), 2 bước cuối Tester thực hiện. Gate sau mỗi phase AI để đảm bảo chất lượng trước khi đi tiếp.

03 · Chi Tiết Từng Bước

5 bước chi tiết

1
Brainstorm Test Strategy với AI
AI + Tester
InputRequirements / User Story / Acceptance Criteria / Design Doc
Hoạt độngTester prompt AI đặt câu hỏi Socratic: scope là gì? user flow nào critical? risk ở đâu? test levels cần cover (unit / integration / E2E)? data & environment cần chuẩn bị?
AI outputDraft Test Strategy: scope in/out, test levels, test types (functional / regression / edge case), priority matrix, risk areas, entry / exit criteria
Tester actionChallenge AI — bổ sung context business, loại bỏ phần không liên quan, confirm scope
OutputTest Strategy Doc (đã được Tester approve)
Gate: Tester approve strategy → tiếp Step 2
2
Plan Viết Testcase với AI
AI + Tester
InputTest Strategy Doc (từ Step 1) + Template Testcase
Hoạt độngAI chia strategy → danh sách testcase cần viết: mỗi nhóm feature có bao nhiêu case, loại gì (happy / unhappy / edge / boundary), priority
AI outputTestcase Backlog — bảng list: ID · Feature · Scenario · Type · Priority · Estimated count
Tester actionReview backlog — bổ sung case thiếu, xoá case trùng, chốt priority
OutputTestcase Writing Plan (approved backlog)
Gate: Tester approve plan → tiếp Step 3
3
AI Viết Testcase Theo Template
AI
InputTestcase Writing Plan + Template chuẩn của công ty
Template fieldsID · Title · Precondition · Test Steps · Test Data · Expected Result · Priority · Type · Related Requirement
Hoạt độngAI viết tuần tự từng testcase theo template — KHÔNG skip field, KHÔNG placeholders, mỗi step phải actionable
AI outputTestcase Suite đầy đủ theo template (Excel / Markdown / TestRail format)
Lưu ýAI phải cite requirement ID cho mỗi testcase (traceability)
4
Tester Review Testcase Theo Tài Liệu
Tester
InputDraft testcase suite + Requirements + Template checklist
Checklist ☐ Đúng template  ·  ☐ Cover đủ requirement (traceability)  ·  ☐ Step rõ ràng, actionable  ·  ☐ Expected result đo lường được  ·  ☐ Không trùng lặp  ·  ☐ Edge case hợp lý  ·  ☐ Test data realistic  ·  ☐ Priority đúng
ActionĐánh dấu PASS / FIX / REJECT từng case. Case FIX → yêu cầu AI viết lại với feedback cụ thể.
LoopNếu > 20% case cần FIX → quay lại Step 3 với prompt refined
OutputTestcase Suite đã approve — production-ready
Gate: 100% case pass checklist → tiếp Step 5
5
Tester Execute Testing Theo Testcase
Tester
InputTestcase Suite đã approve + Build / Environment cần test
Hoạt độngTester chạy từng testcase manual (hoặc semi-auto với tool hỗ trợ), ghi kết quả Pass / Fail, attach evidence (screenshot / log / video)
AI hỗ trợAI giúp: viết bug ticket từ fail case, summarize test report, phân tích pattern của các bug (optional)
OutputTest Report + Bug Tickets (có evidence)

06 · Scope Note

Chưa làm gì trong giai đoạn này

Chưa build agent tự động execute test. Khi team đã thuần quy trình 5 bước này (~6 tháng áp dụng thực tế), sẽ đánh giá lại việc auto-execution, Playwright MCP, Allure integration, flaky detection ở phase sau. (Theo nguyên tắc YAGNI — không build thứ chưa cần.)
