Chi Tiet Tung Phase — Greenfield vs Brownfield
1
Phase 1: Analysis Discovery — Hieu bai toan
Analyst PM
▼
Greenfield — Du an moi
  • Brainstorming & Project Brief Analyst
    Analyst agent dan dat brainstorming session: muc tieu, stakeholders, problem statement, success criteria. Output: project brief.
  • Scope Definition PM
    PM agent define scope va constraints: MVP boundaries, out-of-scope items, timeline assumptions. Bat dau tu mot to giay trang.
  • Context Documentation DOC
    Output: project-brief.md, context.md. Ngan gon, tap trung vao "what" va "why", chua "how".
Brownfield — Du an cu
  • MANDATORY: Document Existing System BROWN GATE
    Buoc bat buoc: truoc khi lam bat ky thu gi, phai document codebase. Skip buoc nay se gay ra documentation debt khong the phuc hoi.
  • Analyst Scan Codebase Analyst
    Analyst doc source tree, identify tech stack, map modules. Tao source-tree-analysis.md. Interview stakeholders de hieu business context.
  • Choose Strategy: PRD-First vs Document-First BROWN
    Large codebase: PRD-first (chi document areas lien quan feature moi). Small codebase: Document-first (doc het roi moi viet PRD).
2
Phase 2: Planning PRD & User Stories
PM SM UX
▼
Greenfield — Du an moi
  • PRD Generation (15-25 pages) PM DOC
    PM agent tao Product Requirements Document: features, user personas, business goals, success metrics, non-functional requirements.
  • Epic & User Story Breakdown SM
    Scrum Master breakdown PRD thanh epics → user stories. Moi story co acceptance criteria ro rang, size-able trong 1 sprint.
  • UX Specs & Mockups UX DOC
    UX Designer tao wireframes, user flows, design system. Output: ux-spec.md voi screens va interactions.
Brownfield — Du an cu
  • Brownfield PRD Template BROWN DOC
    Dung brownfield-prd-tmpl: template co san section cho integration points, existing features affected, migration concerns.
  • Incremental Stories SM
    Stories nho hon binh thuong. Moi story chi touch 1-2 modules. Co explicit section: "code areas affected", "tests to update".
  • Integration Specs BROWN
    Moi story co integration spec: API contracts giu nguyen hay thay doi? Database migrations can khong? Backward compatibility policy?
3
Phase 3: Solutioning Architecture & Test Strategy
Architect TEA
▼
Greenfield — Du an moi
  • Architecture Design Architect DOC
    Architect thiet ke system tu dau: components, data flow, tech stack choice, ASR (Architecturally Significant Requirements).
  • Test Strategy & Test Design TEA DOC
    Test Architect (TEA) tao test-design-architecture.md, test-design-epic-N.md. Risk-based: uu tien test cho critical paths.
  • Sprint Planning Architect SM
    Architect va SM align technical sequence voi business priorities. Dependencies giua stories duoc identify.
Brownfield — Du an cu
  • Brownfield Architecture Template BROWN DOC
    Dung brownfield-architecture-tmpl: focus vao delta giua hien tai va tuong lai. Khong lam lai architecture toan bo.
  • Regression Test Strategy TEA BROWN
    TEA uu tien regression tests: identify breaking change points, viet test cho existing behavior truoc khi thay doi. Legacy dependency risks duoc flag.
  • Migration & Rollback Plan Architect BROWN
    Architect tao migration plan chi tiet: steps, order, feature flags. Rollback plan bat buoc cho moi breaking change.
4
Phase 4: Implementation Code, Test, Quality Gates
Developer QA
▼
Greenfield — Du an moi
  • Dev-Story Format Developer
    Developer agent implement theo dev-story format: moi story co self-contained spec, acceptance criteria, test plan.
  • Automated Test Suite QA
    QA agent chay full test suite moi story: unit + integration + e2e. Test pass la precondition cho quality gate.
  • Quality Gate: Pass/Fail/Defer GATE
    TEA danh gia: PASS (ship), FAIL (fix), DEFER (ship voi known debt). Quyet dinh duoc document.
Brownfield — Du an cu
  • Regression-First Implementation Developer BROWN
    Developer chay regression tests TRUOC khi bat dau story. Bat ky test nao fail ngay lap tuc = stop, investigate truoc.
  • Feature Flags & Gradual Rollout BROWN
    Moi breaking change ship behind feature flag. Gradual rollout: internal → beta users → 10% → 100%. Monitor metrics sau moi stage.
  • Extended Quality Gate GATE QA
    Quality gate check them: regression suite pass? performance khong giam? backward compat verified? rollback tested? Cao hon greenfield.

Nguyen Tac Cot Loi Cua BMAD
AI La Thinking Partner
AI khong tu sinh code. AI la specialist expert ho tro quyet dinh o moi phase. Human drive process.
Quality Gates Giua Phases
Moi phase co checkpoint validation. Artifact khong pass gate thi khong sang phase ke tiep. Prevent downstream corruption.
Specialized Agent Roles
12+ agents voi input/output ro rang. "Agent-as-Code" — dinh nghia trong markdown files, reproducible.
Scale-Adaptive
Workflow tu dong dieu chinh do phuc tap: bug fix = nhe, enterprise = full pipeline. Khong one-size-fits-all.
Brownfield-First Thinking
BMAD khac biet: co workflow rieng cho legacy. Documentation phase la bat buoc, khong optional.
Test Architecture (TEA)
Risk-based testing tich hop natively. Regression & NFR duoc consider tu phase Solutioning, khong phai afterthought.
Artifact-Driven
Moi phase co output files cu the (PRD.md, architecture.md...). Auditable, versionable, handover-ready.
Party Mode
Multi-agent collaborative session: PM + Architect + TEA cung discuss mot van de. Cross-functional decisions.
